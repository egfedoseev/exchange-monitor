# Crypto Arbitrage Simulator (Exchange Monitor)

Пет-проект, представляющий собой симулятор межбиржевого и внутрибиржевого криптовалютного арбитража. Система отслеживает расхождения в курсах валютных пар, анализирует потенциальную прибыль с учетом торговых комиссий и симулирует исполнение ордеров в реальном времени.

Основная цель проекта — построение расширяемой, отказоустойчивой и событийно-ориентированной архитектуры для мониторинга высоконагруженных потоков данных.

---

## Архитектурные решения и фокус на технологиях

Этот проект спроектирован с упором на лучшие практики бэкенд-разработки на Kotlin и принципы SOLID. Ниже приведены ключевые решения, на которые стоит обратить внимание при ревью кода:

### 1. Инкапсуляция инфраструктуры и инверсия зависимостей (DIP / ISP)
Бизнес-логика поиска арбитражных окон полностью изолирована от инфраструктурного слоя.
* Доменные интерфейсы `Exchange` и `Wallet` описывают абстрактное поведение торговой площадки и кошелька.
* Благодаря этому ядро системы (`Analyzer` и `ExecutionManager`) работает исключительно с абстракциями. Это позволяет бесшовно переключаться между симуляционным контуром (`VirtualExchange`, `VirtualWallet`) и интеграцией с реальными биржами по API (слот под `Binance`). Разработка интеграции с новыми биржами не требует изменения логики поиска связок.

### 2. Реактивный конвейер данных (Kotlin Coroutines & Flows)
Рыночные апдейты представляют собой непрерывный и высокоинтенсивный поток событий.
* Вместо блокирующих вызовов (Thread-per-request) или громоздких коллбэков, обработка событий `TradeEvent` реализована на базе **Kotlin Coroutines** и асинхронных потоков **SharedFlow / Channels**.
* Это обеспечивает неблокирующий ввод-вывод (Non-blocking I/O) и минимальные задержки при передаче стаканов от сетевого слоя к аналитическому движку, что критически важно для систем высокочастотной торговли.

### 3. Безопасность типов и выразительность доменной модели
Проект активно использует синтаксический сахар и систему типов Kotlin для предотвращения ошибок на этапе компиляции:
* **Sealed-иерархии:** Потоковые события `TradeEvent` спроектированы через `sealed class` / `sealed interface`. Это гарантирует безопасность и исчерпываемость (`exhaustive when`) при их обработке аналитиком.
* **Иммутабельность:** Все промежуточные состояния данных и события обмена представлены в виде неизменяемых `data class`, что исключает появление багов, связанных с состоянием гонки (Race Conditions) в многопоточной среде.
* **Точность вычислений:** Балансы кошельков (`Wallet`) и расчеты профита защищены от деградации точности из-за специфики типов с плавающей точкой (IEEE 754), обеспечивая строгий учет активов и комиссий.

## Структура проекта

```text
src/main/kotlin/exchange/
├── Main.kt                      # Точка входа, инициализация контекста приложения
├── Exchange.kt                  # Базовый интерфейс биржи (управление подключениями, ордерами)
├── VirtualExchange.kt           # Эмулятор биржи для бэктестинга стратегий без реальных средств
├── Binance.kt                   # Контур интеграции с API/WebSocket биржи Binance (в разработке)
│
├── analyzer/
│   ├── Analyzer.kt              # Движок поиска арбитражных ситуаций (математический обсчет спредов)
│   ├── ExecutionManager.kt      # Менеджер конкурентного контроля и отслеживания жизненного цикла ордеров
│   └── TradeEvent.kt            # Модель событий рынка (типы ордеров, изменения цен, апдейты стаканов)
│
└── wallet/
├── Wallet.kt                # Абстракция управления балансами и валидации активов
└── VirtualWallet.kt         # Локальный кошелек, учитывающий комиссии и проскальзывания симуляции
```

## План развития (Roadmap)

- [ ] Полноценная реализация сетевого клиента `Binance` на базе WebSockets / Ktor HTTP Client.
- [ ] Интеграция API-клиента `Bybit` для полноценного межбиржевого (Cross-exchange) арбитража.
- [ ] Реализация поиска трехстороннего (треугольного) внутрибиржевого арбитража.
- [ ] Покрытие бизнес-логики анализа и управления ордерами модульными тестами с использованием `MockK` и `Kotest`.
- [ ] Логирование метрик и производительности через Micrometer / Prometheus.

## Требования и запуск

* **Стек:** Kotlin 1.9+, Gradle (Kotlin DSL).
* Проект собирается стандартной командой:
  ./gradlew build
* Запуск симулятора:
  ./gradlew run
