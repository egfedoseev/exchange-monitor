# Exchange Monitor (Crypto Arbitrage Simulator)

Пет-проект для симуляции межбиржевого арбитража. В реальном времени следит за курсами на разных биржах (через веб-сокеты), считает потенциальный спред с учетом комиссий и симулирует встречные сделки (купи на одной, продай на другой).

Делал в основном для себя — попрактиковаться с Kotlin Coroutines, реактивными потоками (Flow) и Ktor.

---

## Что под капотом (Стек)

* **Kotlin & Coroutines / Flow** — вся реактивная обработка котировок крутится на корутинах.
* **Ktor (3.5.0)** — используется и как клиент (для веб-сокетов Binance/Bybit), и как сервер (REST API + Swagger UI).
* **ktoml** — для сохранения настроек и балансов в человекочитаемом `config.toml`.
* **BigDecimal** — все денежные расчеты и балансы считаются с фиксированной точностью (никаких `Double` с их погрешностями округления).
* **Logback** — логирование сделок в `logs/trades.log`.

---

## Архитектура и интересные моменты

1. **Реактивные сокеты с автореконнектом**  
   Для получения цен используются WebSocket-клиенты: Binance (`bookTicker`) и Bybit (`orderbook.1`). В базовом клиенте `AbstractExchangeClient` настроен автоматический пинг (каждые 20 секунд) и переподключение при обрыве связи — подписки накатываются заново сами.

2. **Абстракции**  
   Бизнес-логика ничего не знает о реальных биржах и кошельках. Всё работает через интерфейсы `Exchange` и `Wallet`. Это позволяет на лету подменять реальные коннекторы симуляторами (`VirtualExchange`, `VirtualWallet`) для безопасного тестирования логики.

3. **Точный учет комиссий**  
   В симуляции учитывается реальная комиссия биржи (`feeRate`, по дефолту 0.1%). Комиссия вычитается из результирующего актива: при покупке — из базового, при продаже — из котируемого.

4. **Глобальный JSON-сериализатор**  
   Для Ktor настроен кастомный сериализатор `BigDecimal`, чтобы API корректно отдавало балансы и доходность (например, в эндпоинте `/analyzers/profit`).

## API и документация

При запуске приложения автоматически поднимается Swagger. Посмотреть доступные эндпоинты и выполнить запросы можно здесь:  
`http://localhost:8080/swagger`

Основные эндпоинты:
* `GET /exchanges` — список доступных бирж.
* `GET /wallets` / `POST /wallets` — балансы и добавление виртуальных кошельков.
* `GET /analyzers` / `POST /analyzers` — управление запущенными анализаторами.
* `GET /analyzers/profit` — статистика по заработанной прибыли.

## Как запустить

Для работы нужна Java 21+.

1. **Сборка проекта:**
   ```bash
   ./gradlew build
   ```
2. **Запуск тестов:**
   ```bash
   ./gradlew test
   ```
3. **Локальный запуск приложения:**
   ```bash
   ./gradlew run
   ```

Конфигурация по умолчанию лежит в `config.toml`. Все изменения (добавление кошельков, запуск новых анализаторов) сохраняются туда же автоматически.
